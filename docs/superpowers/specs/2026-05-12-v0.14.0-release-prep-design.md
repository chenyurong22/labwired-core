# LabWired Core v0.14.0 Release Prep Design

## Goal

Prepare and ship LabWired Core `v0.14.0` as a minor release that captures the substantial hardware-validation, board-support, simulator, and documentation work completed after `v0.13.0`.

## Scope

This release prep updates release metadata and release-facing documentation only. It does not change simulator behavior, delete tracked fixtures, upgrade dependencies for their own sake, or rewrite generated catalog data.

## Versioning

`0.14.0` is a minor bump because the post-`v0.13.0` history includes new capabilities: ESP32-S3 support, STM32L476 and STM32F407 hardware-validation lanes, expanded peripherals, hardware-oracle tooling, and DAP/debugger improvements.

## Documentation Updates

- `Cargo.toml` workspace package version moves from `0.13.0` to `0.14.0`.
- `Cargo.lock` is refreshed by Cargo after the version change.
- `CHANGELOG.md` gets a dated `0.14.0` section, a new empty `Unreleased` section, and concise release notes based on commits since `v0.13.0`.
- `README.md` install snippets pin `LABWIRED_VERSION=v0.14.0`.
- `RELEASE_PROCESS.md` replaces stale concrete `v0.12.0` examples with neutral `vX.Y.Z` examples.
- `RELEASE_READINESS_CHECKLIST.md` becomes a fresh `v0.14.0` checklist rather than a completed `v0.13.0` artifact.
- `ROADMAP.md` and `scripts/install.sh` stop presenting old versions as current examples.

## Cleanup

Tracked generated outputs under `out/**`, `examples/nucleo-h563zi/target/firmware`, `examples/arm-c-hello/src/main.o`, and the zero-byte `examples/hil-displacement-showcase/hil_displacement_showcase.bin` are removed from source control. Intentional test fixtures under `tests/fixtures`, `crates/core/tests/fixtures`, `fixtures/xtensa-asm`, and golden-reference directories are preserved.

## Release Publication

After verification passes, commit the release prep changes, create an annotated `v0.14.0` tag, push `main` and the tag, and create a GitHub release using the `CHANGELOG.md` `0.14.0` notes.

## Verification

Run:

- `cargo check`
- `cargo fmt --all -- --check`
- `cargo test --workspace`
- `rg -n 'v0\\.13\\.0|0\\.13\\.0|v0\\.12\\.0|0\\.12\\.0' README.md RELEASE_PROCESS.md RELEASE_READINESS_CHECKLIST.md ROADMAP.md scripts/install.sh Cargo.toml CHANGELOG.md`

The final grep may still show historical changelog entries for old releases. Release-facing current instructions must not point to stale versions.
